# grappa Phase 2 — auth + multi-user + per-network credentials

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Multi-user grappa server. Users in DB (Argon2 password hash), opaque-session-ID bearer tokens with sliding-7d-idle expiry, per-network credentials (Cloak.Ecto encrypted at rest with env key), real upstream auth state machine (SASL + Bahamut PASS-handoff + NickServ IDENTIFY fallback + open networks), per-user scrollback isolation, irssi-shape `network → many servers` schema. `grappa.toml` deleted entirely; users + networks managed via mix tasks.

**Design pre-work (S20 design pass):** all seven decisions A–G locked in conversation before plan-write. Recap below; rationale lives inline at each sub-task.

| Decision | Outcome | Why |
|----------|---------|-----|
| **A. Token model** | Opaque session ID, sliding 7d idle | True revocation, "active sessions" UI free, JWT footguns avoided; sliding gives "active forever / idle 7d" semantics user wanted |
| **B. NickServ-pwd at rest** | Cloak.Ecto AES-256-GCM, env key | Encryption-at-rest only (passive disk theft); e2e privacy = OTR in cicchetto, not server's job |
| **C. Users source** | DB only via `mix grappa.create_user`; `grappa.toml` deleted | Mix task is the primitive; no env-seed defaults (anti-pattern); deploy ergonomics deferred until shellless-deploy is real |
| **D. NickServ REGISTER proxy** | Defer to Phase 5 | Network-specific reply parsing, rare op, needs general request/response correlation machinery (also for SETSTATUS/GHOST/RECOVER) |
| **E. Upstream auth flow** | Per-network `auth_method`: `auto / sasl / server_pass / nickserv_identify / none` | Azzurra (Bahamut) confirmed via source: PASS at register triggers server-side `SIDENTIFY` to NickServ; not SASL-capable. State machine handles all paths; auto-mode just merges sasl+server_pass |
| **F. Schema** | Irssi-shape `networks` (logical) → `network_servers` (physical, many-per-net) → `network_credentials` (per-user) | `networks.id integer + slug unique` (text PK feels off, integer cheap); irssi split anticipates multi-server failover (Phase 5 logic, schema-ready Phase 2) |
| **G. Per-user iso on messages** | Add `messages.user_id` UUID FK; PubSub topic shape `grappa:user:{user}/network:{net}/channel:{chan}` | Per-user scrollback (each Session writes own rows from its wire view); topic must encode user_id (PubSub topics are global namespaces, no per-socket filtering); wire shape OMITS user_id (client knows from `/me`) |

## Scope

**In scope (this plan):**

- Users: DB-backed, Argon2-hashed passwords, mix-task-managed.
- Sessions: opaque UUID bearer tokens with sliding 7d idle expiry + revocation table.
- Auth REST: `POST /auth/login`, `POST /auth/logout`, `GET /me`. All other endpoints behind authn pipeline.
- Channel auth: `connect/3` token verify; per-topic authz (`socket.assigns.user_id == topic_user`).
- Networks/servers/credentials: irssi-shape schema, Cloak.Ecto-encrypted upstream passwords, mix tasks for CRUD.
- IRC.Client: real CAP LS 302 negotiation; SASL PLAIN flow; PASS-at-register; NickServ IDENTIFY post-001; auto-detection.
- Session.Server: loads credentials from DB; passes correct auth flow to IRC.Client based on `auth_method`.
- `messages` table: per-user `user_id` FK + new index.
- PubSub topic shape: `grappa:user:{user}/network:{net}/channel:{chan}` (BREAKING vs Phase 1).
- Bootstrap: reads `users` from DB, iterates their `network_credentials`, spawns one Session per binding.
- `grappa.toml` deleted. `Grappa.Config` module deleted. `Grappa.Bootstrap` rewritten.
- Live deploy on Pi against real Azzurra (PASS path), real Libera (SASL path) for end-to-end validation.

**Out of scope (Phase 5+ hardening):**

- NickServ REGISTER proxy + reply correlation machinery.
- Multi-server failover logic (schema is ready; logic deferred).
- Post-registration `+r` umode check + IDENTIFY-as-fallback retry (when PASS handoff silently fails).
- NickServ NOTICE reply parsing (success/fail detection, GHOST/RECOVER for nick collisions).
- Reconnect/backoff policy on upstream drop.
- "Active sessions" UI (sessions table is ready; UI can land Phase 4).
- HSM-keyed Cloak.Vault (operator can swap key source via Cloak config).
- Forced password rotation flow (no env-seed = no force-rotate trigger Phase 2).
- Open registration mode (REST `POST /auth/register` is admin-only Phase 2; public-self-service is Phase 3+).
- OAuth/OIDC providers (opaque-session-ID model is provider-agnostic; trivially additive later).
- Scrollback eviction policy.
- Phoenix Token signing salt → runtime.exs (carry-over from Phase 1.5 todo, lands in this phase since session-token signing-salt lives here).

## Plan-deviation discipline

**Re-plan against current code at every sub-task.** Phase 1 walking-skeleton accumulated 14 deviations before S9 fresh-replan; CP06 S19 Boundary work caught 3 more in code review. The active checkpoint and CLAUDE.md are authoritative when this plan and the codebase disagree — flag the divergence to vjt and re-anchor the plan to the code, not the reverse.

CLAUDE.md rules that bite hardest in Phase 2 work:

- **No `Application.get_env` outside `config/`** — Cloak.Vault config + Phoenix.Token signing salt both read env at boot in `application.ex` and are passed via `start_link` opts.
- **Atoms or `@type t :: literal | literal` — never untyped strings** — `auth_method`, message kinds, etc. all live as atom enums in code, validated at the schema boundary.
- **No default arguments via `\\`** — every new function parameter is required. Especially relevant for `Sessions.create_session/3` (user_id, ip, ua all required).
- **Let it crash** — SASL 904/905 = abort connect, let supervisor restart. No try/rescue around state-machine branches.
- **Phoenix Channels = the event push surface, not state changes** — auth REST endpoints (login/logout/me) are REST. Don't be tempted to put login over a Channel.
- **Schema future-proofing** — every column added now must work for both Phase 2 and the Phase 5 hardening pile. `password_encrypted bytea null` (nullable for `auth_method='none'`); `auth_command_template text null` (rare per-network override); `realname text null` (defaults to nick if absent).

4. `Accounts` context additions:
   ```elixir
   @idle_timeout_seconds 7 * 24 * 3600
   @last_seen_bump_threshold_seconds 60

   @spec create_session(Ecto.UUID.t(), String.t() | nil, String.t() | nil) ::
           {:ok, Session.t()}
   def create_session(user_id, ip, user_agent) do
     now = DateTime.utc_now()
     %Session{}
     |> change(%{
       user_id: user_id,
       created_at: now,
       last_seen_at: now,
       ip: ip,
       user_agent: user_agent
     })
     |> Repo.insert()
   end

   @spec authenticate(String.t()) ::
           {:ok, Session.t()} |
           {:error, :invalid_token | :not_found | :revoked | :expired}
   def authenticate(token) do
     with {:ok, _uuid} <- Ecto.UUID.cast(token),
          %Session{revoked_at: nil} = session <- Repo.get(Session, token) do
       check_idle(session)
     else
       :error -> {:error, :invalid_token}
       nil -> {:error, :not_found}
       %Session{revoked_at: _} -> {:error, :revoked}
     end
   end

   defp check_idle(session) do
     now = DateTime.utc_now()
     idle = DateTime.diff(now, session.last_seen_at, :second)
     cond do
       idle > @idle_timeout_seconds -> {:error, :expired}
       idle > @last_seen_bump_threshold_seconds -> {:ok, touch_session(session, now)}
       true -> {:ok, session}
     end
   end

   defp touch_session(session, now) do
     {:ok, updated} = session |> change(last_seen_at: now) |> Repo.update()
     updated
   end

   @spec revoke_session(Ecto.UUID.t()) :: :ok
   def revoke_session(id) do
     from(s in Session, where: s.id == ^id, update: [set: [revoked_at: ^DateTime.utc_now()]])
     |> Repo.update_all([])
     :ok
   end
   ```

   ```elixir
   def call(conn, _opts) do
     case get_token(conn) do

           {:ok, session} ->
             conn
             |> assign(:current_user_id, session.user_id)
             |> assign(:current_session_id, session.id)
           {:error, _reason} ->

     end
   end

   defp unauthorized(conn) do
     conn
     |> put_resp_content_type("application/json")
     |> send_resp(401, ~s({"error":"unauthorized"}))

   end
   ```
