# BUGHUNT-2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop the unread marker from vanishing within ~500ms of opening a window, and rewrite the cursor-write contract so all three triggers (switch / scroll-settle / blur) use the honest `lastFullyVisibleRowId` value with the bug fix attaching the input-event gate to scroll-settle so programmatic activation scrolls do not falsely advance the cursor.

**Architecture:** Move cursor-write ownership FROM `cicchetto/src/lib/selection.ts` TO `cicchetto/src/ScrollbackPane.tsx` (the pane owns its DOM geometry; selection.ts owns "which window is focused"). ScrollbackPane fires `setCursorIfAdvances` from four seams: (1) `onScroll` settle debounce, gated by a recent real input event; (2) `on(key, …)` effect's leave-arm BEFORE activation routine touches listRef; (3) `onCleanup` for the unmount-switch case (channel→home/mentions); (4) `on(isDocumentVisible, …)` true→false transition. selection.ts's two cursor-write blocks are deleted; `setCursorForWindow` helper is deleted.

**Tech Stack:** SolidJS + TypeScript 6 (cic side), Playwright (e2e), Vitest + jsdom (unit). No server changes.

**Spec:** `docs/superpowers/specs/2026-05-24-bughunt-2-cursor-design.md`
