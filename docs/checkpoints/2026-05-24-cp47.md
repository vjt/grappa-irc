### Cache "mystery" debunked

There was no cache bug. The grep methodology in the pre-clear
checkpoint was wrong: vite/esbuild mangles the `visibleTailSnapshot`
identifier to a 1-letter local in minified output, so
`grep -c "visibleTailSnapshot"` returned 0 even though the code was
present. Smoking-gun proof: injecting
`console.log("CACHE_PROBE_B7_SENTINEL_XYZZY42")` shipped a NEW bundle
(`DTUf-S5r`), removing it returned to the B7 hash (`BE5RP0Ub`). HEAD
without the B7 stash built to a THIRD hash (`C-Ph5y4M`, the prod hash
from `a2678f7`). Sourcemap `sourcesContent[107]` for ScrollbackPane.tsx
contains all 5 `visibleTailSnapshot` references — bundle is honest,
the verification approach was not.

Learning: when verifying that minified output contains a source change,
search for a literal that survives minification (a string, a comment
in a `/*! */` block, a non-mangled module export) — never an identifier
that the minifier is free to rename.

## CIFIX — Elixir CI flake LANDED (2026-05-24, post-cp47-push)

`origin/main 8b5b6dd → 237819f`. Three commits, three real bugs
behind a single CI failure pattern (run `26371004010`):
`AdmissionStateHelpers.reset_session_supervisor` 15s registry-drain
timeout caused by orphan `Session.Server` crashing on
`Process.exit(pid, :shutdown)` from the helper.

* **`0905757` — `session: relax M7 EXIT raise to OTP-convention
  stop`.** The REV-J catch-all at `handle_info({:EXIT, _,
  :shutdown | :normal}, _)` raised unconditionally; the mailbox
  tuple cannot distinguish a linked-process clean exit (the
  design violation the raise was meant to surface) from an
  external `Process.exit/2` (a standard OTP teardown primitive).
  Replaced with `{:stop, reason, %{state | client: nil}}` — the
  OTP convention. `:shutdown` / `:normal` are non-restart reasons
  for `:transient` children so no spurious respawn. Future
  `Process.link/1` sites within the module must add an explicit
  earlier clause if they need bespoke handling (comment block
  documents the rationale).
* **`aa47197` — `test/admission: GenServer.stop for orphan-Session
  cleanup`.** The Registry sweep used `Process.exit(pid,
  :shutdown)` then waited for `:DOWN`; replaced with
  `GenServer.stop(pid, :shutdown, 2_000)` (synchronous, runs
  `terminate/2` deterministically, bypasses the
  mailbox-as-signal path). Drops the manual monitor +
  receive-`DOWN` boilerplate.
* **`237819f` — `test/channels: on_exit Session.stop_session for
  both setup helpers`.** `grappa_channel_test.exs`'s two session-
  spawning helpers had no `on_exit` — sessions outlived the test,
  crashed post-fake-IRC-shutdown with `:econnrefused`,
  `:transient`-respawned, then raced the next test's
  `reset_all/0`. Installed `Session.stop_session/2` teardown per
  helper (the production teardown verb, supervisor-mediated,
  registry-drained).
