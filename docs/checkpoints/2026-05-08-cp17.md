## Design summary

* Broadcast on `Topic.user(...)`, NOT per-channel topic — chicken-and-
  egg: cic only subscribes to per-channel after seeing `:pending`.
  User-topic is joined from boot (`userTopic.ts` createRoot effect).
* New verb: `Grappa.Session.Wire.window_pending(network_slug, channel)`
  → `%{kind: "window_pending", network: ..., channel: ..., state: "pending"}`.
* Snapshot push for `:pending` is INTENTIONALLY a no-op (in-flight
  TTL is 30s; cic snapshot path is per-channel which doesn't apply).
* cic-side: `setPending` export STAYS but is now fed by the user-
  topic dispatcher. The pre-subscribe loop in `subscribe.ts:425`
  re-runs on the signal change regardless of who calls setPending.
