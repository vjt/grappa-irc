**#12 — `/msg` to a channel-shaped target → rejected.** Shipped + deployed
+ closed.

- **Fix (`4f0ddb6`):** `cicchetto/src/lib/slashCommands.ts` `msg:` parser
  now rejects every IRC channel sigil (`# & ! +`, per `channelKey.ts`) up
  front with `err(verb, "/msg to a channel is not supported")`. `err()` →
  `kind:"error"` → `compose.ts` `case "error"` (returns `{error}`)
  short-circuits before `case "msg"`, so no phantom query window opens.
  Services shortcuts (`/ns` `/cs` …) unaffected (separate path, non-sigil
  targets).
- **Root cause:** `case "msg"` routed `/msg #x` through the QUERY path
  (`openQueryWindowState` keyed by a *channel* name); cic's WS-driven
  own-send render only feeds JOINED channel topics, so a `"#x"` query
  window never rendered the send — phantom window, live + post-restore.
- **Decision (vjt):** cic-only reject. Server `send_privmsg` left as-is
  (still sends+persists for a non-joined channel); cic just never
  originates the channel-shaped `/msg`. Same user-visible result as a
  server-side reject, ~4× less code on a single-client system.
- **TDD:** `slashCommands.test.ts` — rewrote the old "spaces preserved"
  test (was asserting the buggy `/msg #italia` → `kind:"msg"`) to a nick

  reject test over the full sigil class. RED (4 fail) → GREEN (128 pass).
