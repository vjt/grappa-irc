## Why N-1+N-2 unified but N-3 separate

N-1 (joined silent) + N-2 (non-joined → `$server`) ARE the same
code path with different inputs. The fix touches one drain
function plus one accumulator. Splitting into two commits would
be theatre. N-3 is in cic only with no server overlap; logically
separate.

## Disarm pattern (N-3)

The `coldLoadAutoSelected` latch is the load-bearing part. Without
it, after self-PART:

- selection rolls to null (subscribe.ts BUG5a)
- effect re-fires, finds another joined channel
- setSelectedChannel(other-joined-channel)
- m9 / cic-members-panel-scope tests fail (they assert empty
  stub holds)

The latch fires once, then refuses forever. Reuse this shape for
similar "fix initial state without fighting subsequent
transitions" effects.
