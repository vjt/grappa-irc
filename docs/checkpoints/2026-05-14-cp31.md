the runtime allowlist fails CI loudly, not a Playwright run.

### Code-reviewer agent caught a latent prod-corruption bug

The reviewer agent flagged a missing `read_cursors` recreate in
B6.11's migration: sqlite >=3.25 auto-rewrites dependent FK refs
during `ALTER TABLE messages RENAME TO messages_old`, then dangles
after the DROP. Same disease class as the 2026-05-04 precedent's
`network_servers` FK. Fix: recreate `read_cursors` in the same
migration. Caught BEFORE landing — exactly the kind of cross-table
issue serial-grep + self-review would have missed. New memory
`feedback_subagent_driven_development` codifies the lesson.

### Lessons captured this cluster

1. **Subagent-driven development for cluster work**
   (`feedback_subagent_driven_development`): the code-reviewer agent
   caught the dangling `read_cursors` FK ref BEFORE landing — exactly
   why per-bucket reviewer loops + Plan-agent design beat linear
   single-thread mode. Pre-loaded orchestrator plans don't excuse
   skipping the agents.

2. **Wire-edge runtime allowlists must be exhaustiveness-tested**: a
   TypeScript discriminated union is compile-time, not runtime.
   `VALID_MESSAGE_KINDS` not having `"server_event"` was a textbook
   silent-drop bug in code shipped to close silent-drop bugs.
   Mitigation: vitest exhaustiveness pin in `wireNarrow.test.ts`.

3. **Sqlite table-recreate has FK-rewrite cascade**: any
   recreate-with-CHECK must recreate every dependent FK-referencing
   table too. Precedent from 2026-05-04 (`network_servers`) repeats
   for B6.11 (`read_cursors` post-2026-05-13). Document the cascade
   list in the @moduledoc — the M5 fragility flag pattern works.
