### Headline

The B1 catch-all closes a real silent-drop class but introduces three
secondary failure modes:

1. **CRIT-1.** B1 catch-all persists `AUTHENTICATE` continuation
   payloads to `$server` scrollback as plaintext — credential-leak
   path; same disease class as the W12 NickServ-leak hardening.
2. **HIGH-2.** Empty-trailing verbs (bare `WALLOPS`, terminal `ERROR`,
   nick-rejected 432 with no trailing) silently dropped via
   `validate_required(:body)` — the very thing B1 was supposed to fix.
3. **HIGH-7.** Catch-all reuses `:notice` kind for non-notice events
   (KILL/WALLOPS/CHGHOST/INVITE) — undermines closed-set discipline.

All three are small diffs; B6 should land them.
