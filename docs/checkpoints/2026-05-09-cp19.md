### Option A — T32 parked-window e2e (HIGH)

Smallest scoped, but blocked on a server-side design pass.
`cp15-b6-parked.spec.ts` does not exist; CP15 B6's brief that it was
"mechanically authorable" was wrong. T32 `Networks.disconnect/2`
terminates the entire `Session.Server` → no per-window `:parked` event
ever produced. Decisions needed before authoring:
1. Do per-channel windows survive park? If yes, Session.Server needs a
   `terminate/2` callback that emits `:parked` on each tracked channel
   before exiting (or the disconnect path moves out of `terminate/2`
   and into a soft-park GenServer call).
2. Is there a per-network "parked" overlay state in cic
   (network-row greying vs per-channel greying)? Needs UX call.
3. When does Session.Server wake on `Networks.connect/1`? Bootstrap
   restart latency vs eager spawn?

The per-network `connection_state_changed` is already reachable in cic
over WS (CP15-codebase-review-fixes H1). The network-overlay path is
trivially wired; only per-window state is the open question.

**Sized**: 2-3 sessions (1 design pass + 1 server-side + 1 cic + e2e).
