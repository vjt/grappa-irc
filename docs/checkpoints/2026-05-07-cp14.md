## Buckets (in flight order)

### B1 — initial scroll position when no fresh unreads
**Symptom**: window with N lines opens mid-scrollback (anchored to
unread-marker DOM ref) instead of at bottom, even when read cursor
is caught up.
**Root cause**: `cicchetto/src/ScrollbackPane.tsx:577-598` createEffect
on messages-length anchors to markerRef whenever it exists — but the
marker exists even when cursor is current (cursor row is "at or after
itself").
**Fix**: only anchor to marker when ≥1 message exists STRICTLY AFTER
the marker row's `server_time`; else tail-follow.
**Tests**: vitest (RED first); browser smoke at close.
**Status**: not started.

### B2 — scroll-up doesn't loadMore
**Symptom**: server has hundreds of older rows, cic stops at oldest
loaded, no fetch on scroll-to-top.
**Root cause**: `cicchetto/src/ScrollbackPane.tsx:611-615` onScroll
only updates `atBottom`; never calls `loadMore` (which exists at
`cicchetto/src/lib/scrollback.ts:173`).
**Fix**: onScroll → if `scrollTop ≤ 200px` AND no in-flight loadMore
for this key → call `loadMore`. Add per-key in-flight Set in
`lib/scrollback.ts` (the long-deferred concurrency guard from S22
review C5). Preserve scroll position on prepend
(`scrollTop = newH - oldH + oldScrollTop`).
**Tests**: vitest only (e2e: deterministic >50-msg seed too costly,
waiver per handoff).
**Status**: **LANDED 2026-05-07**, deployed to voygrappa.bad.ass
@ 2a1b7b8.

### B3 — DM (query) windows show only outbound
**Symptom**: query window for `<peer>` shows only `vjt-grappa → peer`
messages; `peer → vjt-grappa` history invisible.
**Root cause**: IRC framing — outbound persists on `channel=peer`,
inbound persists on `channel=own_nick`. cic's `loadInitialScrollback(peer)`
only fetches `?channel=peer`. Wrinkle: own-nick rotated
`grappa → vjt-grappa` post-NickServ-regain, so historical inbound is
sharded across past own-nicks.
**Path chosen (vjt 2026-05-07)**: **(c) `:dm_with` schema field** —
migration adds normalized "DM peer" col, backfill at migrate-time,
populated at persist-time on PRIVMSGs whose target=own_nick OR
sender=own_nick. Solves own-nick rotation permanently — no second
endpoint to enumerate historical own-nicks.
**Status**: **LANDED 2026-05-07**, deployed to voygrappa.bad.ass
@ `b55e04e`.

Implementation split across four bite-sized commits:
- `2dde02c db(cp14-b3): migration adds :dm_with column on messages` —
  adds `dm_with :text NULL` + `(network_id, dm_with, server_time)`
  index. Inline backfill for both user (`network_credentials`) and
  visitor (`visitors` JOIN `networks` on slug) credentials. Heuristic:
  current credential nick used to detect inbound (channel == own_nick
  → dm_with = sender) + outbound (sender == own_nick AND target nick-
  shaped → dm_with = channel). Pre-rotation rows under a past own-
  nick stay nil — accepted trade-off; the write-time path is canonical.
- `47866bc scrollback(cp14-b3): :dm_with field + dm_peer/4 + bidirectional fetch` —
  `Grappa.Scrollback.dm_peer/4` is the single source for the DM
  detection rule (case-insensitive own_nick comparison; nick-shaped
  target check); both EventRouter and Session.Server call it so the
  inbound + outbound halves of every exchange land with matching
  `dm_with` values. `fetch/5` rewritten to dispatch via
  `dm_eligible?/1`: nick-shaped names route to
  `channel == ^name OR dm_with == ^name`, channel-shaped + $server
  keep the existing `channel == ^name` filter. 7 new scrollback tests
  (3 persist + 4 fetch).
- `f718f6e session(cp14-b3): persist :dm_with on inbound + outbound PRIVMSGs` —
  wires `Scrollback.dm_peer/4` into EventRouter's `build_persist/6`
  (inbound, has `state.nick` as own_nick) and Session.Server's
  `handle_persisting_send/3` (outbound, `state.nick` is sender +
  target is the param). For non-PRIVMSG/ACTION kinds the helper
  returns nil — channel/presence rows write `dm_with = nil`.
- `b55e04e cic(cp14-b3): drop own-nick query filter + add e2e for bidirectional DM` —
  the `shouldKeepInOwnNickQuery` + `ownNickIfOwnNickQuery` filter in
  `cicchetto/src/lib/scrollback.ts` is dead code now that the server
  is authoritative. DELETED entirely (functions, all 3 call sites in
  `loadInitialScrollback`/`loadMore`/`appendToScrollback`, the
  `networks` import, the 12-test filter coverage in
  `__tests__/scrollback.test.ts`, the orphaned `lib/networks` mock).
  Vitest count drops 580 → 568. New e2e
  `cp14-b3-dm-history-bidirectional.spec.ts` exercises the

  both messages visible in the freshly-opened query window).

> The 50cdd21 + a67e890 fixes for members_seeded are correct for
> cic-initiated JOIN, but NOT for the deploy/WS-reconnect/server-
> bootstrap-autojoin path. Server emits `members_seeded` on 366

> server is authoritative for. Fix the class, not the instance.

Brainstorm CLOSED 2026-05-07 with vjt; 7 decisions pinned:
1. Autojoin same path (no special-case).
2. Window opens optimistically on intent, transitions via events.

   failure messages — better UX).

5. Visitor same lifecycle (verify at impl).
6. Archive UX: per-network section, USER-action-driven boundary
   (PART/×, NOT KICK/disconnect), mixed channels+queries sorted by
   last_activity desc, $server always active, delete deferred.
7. members_seeded after_join push STILL needed alongside (deploy
   reconnect race) — same payload pushed from
   `push_channel_snapshot` instead of from EventRouter.

Closes 3 todo items + members race + ghost-window bug + the
