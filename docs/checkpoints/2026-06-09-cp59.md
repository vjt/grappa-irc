### issue #43 — split logout (`ab61e2a`)

The single "log out" was ambiguous about the bouncer: `auth.logout()`
revokes the web bearer + redirects to `/login` but never touches the
upstream IRC session (by design). It surprised the operator (2026-06-04)
— "logged out" of cic, then watched the IRC session keep filling
scrollback.

**Two affordances for registered users**, gated on
`getSubject()?.kind === "user"`:

- **detach** (testid `detach-btn`) — today's `logout()` relabelled: drop
  the web client, leave IRC connected; reconnecting cic resumes the
  session.
- **quit** (testid `quit-irc-btn`) — a destructive two-tap
  `InlineConfirmButton` (`quit` → `really quit IRC?`) wired to the
  **pre-existing** `quitAll(null)` (`lib/quit.ts`: park every
  `kind === "user"` network via `PATCH /networks/:id
  {connection_state:"parked"}`, then `logout()`). Parked persists across
  restart (Bootstrap skips `:parked`) = the correct "stays off until I
  reconnect" Quit semantic.

**Wiring, not new infra** — `quitAll` already backed the `/quit` compose
verb + the visitor sidebar ×. Server untouched.

**Visitors + the not-yet-loaded `null` subject keep the single
"log out".** Gating on `kind === "user"` (not `!isVisitor()`) keeps the
loading state on the safe single button — a null subject must never
render a one-tap bouncer-kill affordance.

**Disarm-on-close.** The drawer stays mounted across open/close (CSS
`.open` toggle, not `<Show>`), so an armed `quit` would survive a
close→reopen one stray tap from killing the bouncer. A
`createEffect(() => { if (!props.open) setQuitArmed(false) })` disarms on
every close; the armed flag lives in the parent per the
InlineConfirmButton contract. `.settings-quit` CSS variant lifts the
compact admin-row base to the drawer's full-width sizing; the shared
`.confirming` rule supplies the red armed treatment.

**Tests.** 5 new vitest (mocked `quitAll`: detach→logout-not-quitAll,
single-tap arms without firing, two-tap→quitAll, disarm-on-close,
visitor single button) + Playwright `issue43-split-logout.spec.ts`
(real-browser render + arm-guard + disarm). The e2e **deliberately does
NOT fire** the destructive confirm or a real detach — vjt's seeded
token + IRC session are shared suite-wide, so parking it or revoking the
bearer would cascade-fail downstream specs; the `quitAll` composite is
already full-stack covered by `u-4-device-identity-change` +
`ux-4-z-cluster-journey`. `m7-admin-gate` positive twin moved
`log out` → the `detach-btn` testid.
