## S2 ‚Äî design

Trace established: `/away` ‚Üí server replies 306 RPL_NOWAWAY ‚Üí
`NumericRouter` routes `{:server, nil}` ‚Üí `$server` channel ‚Üí cic
`isOperatorActionEcho` returned `true` (kind=notice +
meta.numeric=306) ‚Üí predicate fired ‚Üí no bump ‚Üí no badge ‚Üí S8 RED.

**Boundary error**: CP20 conflated two shapes of "row produced by my
action":
1. Routed to a window the operator already inhabits (or just
   created: ghost DM after `/msg ghost`) ‚Äî IS echo; alerting them is
   a false positive.
2. Routed to `$server` (per-network server-messages window) ‚Äî NOT
   echo: the window EXISTS to surface routed server output (MOTD,
   RPL_NOWAWAY, untargeted NOTICEs, lifecycle events).

The discriminator is the routing **TARGET**, not `meta.numeric`
shape alone. CP20's "yes all" answer to "all numerics or only error?"
was right WITHIN scope (the original 401-ghost case) but wrong
outside it.

**Predicate refinement**: one extra clause: `if (message.channel ===
SERVER_WINDOW_NAME) return false;`. The 401-ghost case still fires
(lands in new ghost-nick query window). The 306-to-$server case
stops firing.

**Refactor sidecar**: `"$server"` literal duplicated 6+ times across
compose.ts, subscribe.ts, Sidebar.tsx, BottomBar.tsx. Adding a 7th
call site (the predicate carve-out) on a magic string violated
CLAUDE.md "Implement once, reuse everywhere." Promoted to
`SERVER_WINDOW_NAME` constant in `cicchetto/src/lib/windowKinds.ts`.

## Lessons

1. **Per-spec verification on CI failures.** `gh run view --log-failed`
   is fast and authoritative. Naming the failed spec by name is one
   `grep` away. Blanket-blaming "testnet flake" without inspection
   pins regressions in main silently ‚Äî exactly what
   `feedback_landed_claim_evidence` warned about, and exactly what
   happened here.
2. **`meta.shape` ‚âÝ "produced by my action."** Wire-shape carries
   *production-site* info; *destination* requires a separate read.
   The CP20 predicate took a shortcut that elided the destination
   axis. Carve-outs on the destination side restored correctness.
3. **Magic strings are infrastructure liabilities.** A 7th call site
   forced the refactor ‚Äî the right time to extract a constant is
   when adding a new use, not "later." The 6 existing call sites
   were already a smell; this fix paid the debt. `SERVER_WINDOW_NAME`
   is now the single source of truth pinning the cic-side string
   against the server-side `{:server, nil}` fanout from
   `Grappa.Session.NumericRouter`.

### Lessons

1. **Audit-summary staleness** is its own failure mode beyond
   pointer staleness. Re-audit code, not memory snapshots, when
   "this is what's left" claims arrive across /clear. Memory updated
   (`feedback_survives_clear_pointer_staleness` corollary).
2. **Re-running check.sh in a tight loop is expensive.** When
   introducing a new IRC verb that touches Client + Session + Wire +
   EventRouter + GrappaChannel + cic, run check.sh ONCE at the end
   with everything already correct. Each Sobelow + Dialyzer + Credo
   --strict pass is ~90s; 5 iterations = 8 minutes of "format then
   recheck" instead of one clean shot. Pre-flight before commit:
   `mix format` + manual credo --strict + dialyzer locally.
3. **Worktree first-run cost is real.** A fresh worktree needs
   `bun install` (60s, node_modules not bind-mounted from main) +
   `git submodule update` (testnet, 30s). Budget 2 minutes upfront
   before any vitest / e2e command works.
4. **`<li onClick>` triggers biome a11y `useKeyWithClickEvents`
   when nested in a non-interactive parent.** Workaround:
   `<li><button class="member-name">...</button></li>`. Lift the
   onClick to the <button>, style the <button> to look like the
   former <li> (transparent bg, no border, font:inherit).
5. **dialyzer `@typep verb` is closed-set.** Adding a new
   `Client.send_X/N` requires extending `@typep verb` in the same
   file or the `reject_invalid_line` clause becomes a contract
   violation. Documented at the typespec.
