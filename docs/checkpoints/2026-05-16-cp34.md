## Reviewer-caught bugs

All caught pre-commit during T-3:

1. **Credentials silent-filter**: `list_credentials_for_all_users/0`
   filters `connection_state == :connected`. Verb claim "every bound
   credential" was false; parked + failed hidden. Fix: new
   `Credentials.list_all_credentials/0`. **Verified live post-deploy**:
   `bin/grappa list-credentials` against prod shows vjt's
   `grappa@azzurra` cred as `state=parked reason=user-disconnect` —
   without this fix the row would have been silently invisible.
2. **Registry match too loose**: `{{:"$1", :"$2", :"$3"}, ...}` matched
   any 3-tuple key. Fixed by pinning `:session` literal (mirror of
   `auth_controller.ex:236` pattern).
3. **delete_visitor success line lied on concurrent-reaper race**:
   distinct "already deleted (concurrent reaper or operator)" log.

## Lessons captured

1. **Phantom-bug descope discipline**: T-A7's `:grappa, :visitors,
   ttl_default: 86_400` + boot-time backfill was based on a stale
   observation. Verified via `scripts/db.sh` against prod (2 visitors,
   0 NULL); read schema (`create_changeset` REQUIRES expires_at +
   "must be in future"); read V7 migration (nullable for IDENTIFIED
   only). Backfill would touch zero rows. Per CLAUDE.md "Challenge the
   spec" + S47 "the spec is not the directions" — confirmed back to
   vjt, descoped, T-3 ships smaller.
2. **NetworkCircuit ETS leak closed (B5 review action)**: 11-session-old
   deferred review finding. Per-test-file `clear_registry_for/1`
   silently exhausted 500ms budget under CI load. Fix in
   `AdmissionStateHelpers.reset_session_supervisor/0` — loud raise on
   Registry-converge timeout instead of silent `:ok`.
3. **Log honesty rule** added to CLAUDE.md (Code-shape rules): fast
   paths state what they observed, not what they did. The pre-T-4
   `bootstrap: no credentials bound` is the archetypal anti-pattern;
   codified so future skip-and-log shortcuts have a doc to violate.
