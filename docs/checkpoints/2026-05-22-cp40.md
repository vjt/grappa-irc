## REV-E LANDED 2026-05-22

**`:ok = Client.send_*` strict-bind regression sweep (H11)** — closes
the lone HIGH from the 2026-05-22 codebase review tagged with the
post-U-cluster boundary-fix drift class.

**Root cause:** U-cluster boundary fix (commit `7bb3caa`, 2026-05-17,
per `feedback_no_silent_drops_closed`) changed `Grappa.IRC.Client`'s
send handling — `transport_send/2` got a nil-socket guard returning
`{:error, :no_socket}` cleanly instead of raising
`FunctionClauseError`. Every `:ok = Client.send_*` strict-bind site
in `Session.Server` became a `MatchError` landmine that would crash
the calling Session.Server the next time a dead-socket SEND fired
(operator's autojoin sequence racing a k-line, NickServ ghost-recovery
mid-flow disconnect, AWAY issued during socket RST, etc.). Crashes
funnel via REV-D's `terminate/2` bookkeeping → Backoff bump →
`:transient` respawn → re-crash on next send — exact crash-loop class
REV-D defends against, just with a different trigger.

**Fix shape (9 sites in `lib/grappa/session/server.ex`):**

Two patterns split by caller intent (per CLAUDE.md "No silent-swallow
at boundaries"):

1. **Propagate path** (caller can surface error):
   - `handle_call({:send_mode, ...})` raw MODE (line ~1037) — switched
     to `{:reply, Client.send_line(...), state}` match-arm reply
     (mirrors the existing `send_nick`/`send_quit` precedent at lines
     838-855).
   - `send_chunked_mode/4` (/op /deop /voice /devoice /ban /unban) —
     `Enum.each` → recursive `flush_mode_chunks/3` collect-or-bail per
     CLAUDE.md. Halts on first `{:error, _}` and propagates the typed
     error to the GrappaChannel caller.

2. **Fire-and-forget path** (in-process state mutation, dead-socket
   means supervisor restart owns reconnect):
   - `apply_effects([{:reply, line} | _], _)` (EventRouter outbound,
     e.g. CTCP VERSION NOTICE reply) — paired `:persist` effect
     runs on the next recursion regardless of reply outcome, so the
     scrollback row lands either way.
   - `flush_lines/2` (ghost-recovery NickServ dance) — `pending_auth`
     stage discarded harmlessly on terminate; next reconnect retries
     GHOST + IDENTIFY from scratch.
   - 5 AWAY-internal sites consolidated into a single
     `maybe_log_send_failure/2` helper covering
     `set_explicit_away_internal/3` (2 clauses),
     `set_auto_away_internal/1`, `unset_away_internal/2` (2 clauses).
     Honest about the limitation: AwayState mutator flips local
     state for the LIVE process (so `away_confirmed` broadcast +
     Mentions aggregation track the window), but a Session crash +
     supervisor restart wipes AwayState — operator must re-issue
     `/away` post-reconnect (UX-7 follow-up could surface a hint).
