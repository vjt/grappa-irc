## REV-E staged

Scope: `:ok = Client.send_*` regression sweep ‚Äî finding **H11** from
the 2026-05-22 codebase review.

Pre-U-cluster the `IRC.Client.handle_call({:send, _}, _, _)` path
raised `MatchError` / `FunctionClauseError` on a dead socket (the
session.terminate's wide `:exit, _` catch absorbed it silently for
weeks). U-cluster boundary fix (commit `7bb3caa`, 2026-05-17) made
`transport_send/2` nil-socket-guard return `{:error, :no_socket}`
and `handle_call` return the tagged tuple cleanly. **But the
upstream call sites that `:ok =`-strict-bind the result are now
landmines** ‚Äî once the live socket dies under any caller's foot,
`:ok = Client.send_quit(...)` (or send_join / send_part /
send_privmsg / etc.) will `MatchError` and crash the calling
process. Crashing a Session.Server bumps Backoff (REV-D funnel) ‚Üí
respawn ‚Üí re-crash on next send ‚Äî exact crash-loop class REV-D
defended against, but with a different trigger.

Audit + fix every `:ok = Client.send_*` call site. Replace with
either `with :ok <- ...` (if the caller has somewhere to surface
the error) or `_ = Client.send_*` + Logger.warning on `{:error, _}`
(if the caller is a fire-and-forget hot path that shouldn't gate
on send success).

**Files (per H11 + grep target):**
- `lib/grappa/session/server.ex` ‚Äî autojoin path (line ~1486-ish
  per REV-D edits) + ghost recovery + others
- `lib/grappa/session/event_router.ex` ‚Äî likely several
- `lib/grappa/session/ns_interceptor.ex` ‚Äî NickServ IDENTIFY path
- `lib/grappa/session/ghost_recovery.ex`
- `lib/grappa_web/channels/grappa_channel.ex` ‚Äî user-initiated
  send paths
- `lib/grappa_web/controllers/*.ex` ‚Äî `Session.send_*` wrappers
  may have strict-binds

Run `grep -rn ':ok = .*\.send_' lib/` first to enumerate; likely
8-12 sites. Some sites that are NOT `Client.send_*` but
`Session.send_*` (the public API wrapper) may also need audit
since the session-level wrapper may itself strict-bind.

**Deploy:** server-only, likely HOT (no state-shape change, no new
modules at boot, no migration, no application.ex). Preflight will
classify accurately UNLESS the empty-diff trap recurs
(`feedback_deploy_preflight_empty_diff_after_merge`) ‚Äî when
