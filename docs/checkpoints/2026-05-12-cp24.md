## Three CRITICAL findings (the porco dio trifecta)

- **C1 — SASL credential leak in pre-handshake phases**
  (`lib/grappa/irc/auth_fsm.ex:232-234`). `AUTHENTICATE +` clause
  matched UNCONDITIONALLY for any phase other than `:registered`.
  Hostile/MitM upstream can elicit operator's SASL credentials before
  SASL has been negotiated. Phase guard missing on `:sasl_pending`.
- **C2 — SQLite `PRAGMA foreign_keys` never enabled in dev/prod**
  (`config/runtime.exs:22-27` + `dev.exs:3-6`). 23 migrations'
  worth of `references(..., on_delete: ...)` are runtime-dead.
  CASCADE on user/visitor delete doesn't fire; `:restrict` guards
  don't restrict. The CP19 `defer_foreign_keys=ON` lesson sat next
  to the missing connection-level toggle.
- **C3 — Visitor WHOIS broken** (`lib/grappa_web/channels/grappa_channel.ex:445-454`).
  Comment explicitly flags the bug + prescribes the fix; implementation
  uses the rejected path. Visitors get `visitor_not_allowed` despite
  documented intent.
