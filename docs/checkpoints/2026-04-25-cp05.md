## S16 (2026-04-25) — Codebase review fix bundle (C1-C5)

S15's review recommended bundling all 5 findings into a single small
worktree. Done in one pass: `phase1-codebase-fixes` branched from local
main, ~30 lines net across 5 files, commit `8bff2b3`, fast-forwarded to
main, deployed.

**C1 (HIGH)** + **C2 (MEDIUM)** — `messages_controller.create/2`:
wrapped `Scrollback.persist_privmsg/4` in `with`; expanded `@spec` to
include `{:error, Ecto.Changeset.t()}`. The `FallbackController`
already had the `Ecto.Changeset` clause (`fallback_controller.ex:29`)
— it just never saw changeset errors because the direct-match crash
beat it. With the wrapper, malformed `network_id` / `channel`
identifiers now flow through fallback as 422 (not raise).

**C3 (MEDIUM)** — `IRC.Client.handle_info/2` catch-all added. Stray
`:DOWN` or any unsolicited mailbox traffic now logs + continues
instead of cascading a `FunctionClauseError` through the link to
Session. Required adding `:unexpected` to the Logger metadata
allowlist in `config/config.exs` — could not reuse `:message` since
Logger reserves it for the `$message` template body. Kept the new
key in the "structured debug context" group, not the per-IRC-event
group, so it doesn't interact with the
`Meta.@known_keys ↔ Logger metadata` drift test.

**C4 (MEDIUM)** — `.env.example` gained `DATABASE_PATH`, `POOL_SIZE`,
`PORT`, `GRAPPA_CONFIG`, `LOG_LEVEL` with placeholders + per-env
defaults. Review listed 4 vars; `PORT` was also undocumented — added
for full symmetry rather than just the four called out (CLAUDE.md
`.env.example ↔ runtime.exs symmetry` rule).

**C5 (LOW)** — `Scrollback.Meta` moduledoc rewritten: claimed
`String.to_existing_atom/1` re-atomization; actual code uses
`Enum.find` against `@known_keys`. Doc now reflects the safer
implementation + explains why (bounded by allowlist size, never
touches global atom table — even better than `to_existing_atom/1`'s
load-order-dependent guarantee).

**Gates**: `scripts/check.sh` green — 5 properties, 180 tests, 0
failures, Dialyzer 0 errors, Credo strict + Sobelow + audits clean.

**Live**: `scripts/deploy.sh` rebuild + restart; bouncer reconnected

**Type-system tightening:** `IRC.Message.command` is now a closed atom
enum + `{:numeric, 0..999}` + `{:unknown, String.t()}` discriminated
shape. Parser atomises against an explicit ~24-token allowlist. The
"atom-table-DoS" reasoning that motivated the original `String.t()`
choice applies to unbounded user content, not the closed protocol
vocabulary; corrected the moduledoc reasoning. Session.Server now
matches `:privmsg`, `:ping`, `{:numeric, 1}` instead of strings.

**Session.Server cohesion fix:** A3 + A4 + A5 + A6 together stripped
five concerns out of the GenServer:
- Wire-framing (NICK/USER/PONG) → `IRC.Client.send_handshake/2` +
  `send_pong/2`
- Wire-shape construction → `Wire.message_event/1`
- Prefix-tuple destructuring → `IRC.Message.sender_nick/1`
- Connection-param storage → only `(user_name, network_id, autojoin,
  client)` retained in state; host/port/tls/nick consumed at init
- `Config.Network` struct coupling → start_opts is a flat primitive map
The state map shrank; Phase 5 SASL/CAP work has a clear seam.

**Identifier validation surfaced one real allowlist drift** — the
A18 test (`Meta.@known_keys` ↔ Logger `:metadata`) caught that
`:new_nick`, `:modes`, `:args` were in the Meta allowlist but missing
from the Logger allowlist; they would have been silently dropped from
log output. Fixed in the same commit.

**Discriminated Config errors** turned the bootstrap warning from
"no config — running web-only" (one message for missing-file /
malformed-TOML / missing-field) into four distinct log lines via
`Config.format_error/1`. Side-effect: A20 (originally deferred to
Phase 5) is now mostly addressed.

**Wire-shape contract verified live in prod:** REST + PubSub broadcast
+ Channel push all route through `Wire.message_event/1` /
`Wire.to_json/1`. Single source of truth held end-to-end through the
refactor; live `#grappa` PRIVMSG round-trips work post-deploy.
