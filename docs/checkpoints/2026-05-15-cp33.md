## The text-only invariant

The cluster's most important artifact is the new CLAUDE.md rule
under Engineering Standards → Code-shape rules: **"IRC stays text
only."** No inline rendering of media types in scrollback. URLs
are clickable links via the existing `linkify` path. The
image-upload cluster ships a 📸-prefixed URL pattern that is text
on the wire and a clickable link in cic — that is the model.

The first brainstorm v1 violated the invariant by proposing inline
thumbnails + lightbox. vjt rejected with "IRC REMAINS TEXT
FUCKING ONLY" and the v3 brainstorm halved the bucket count by
honoring the text-only rule end-to-end. Codifying the rule in
CLAUDE.md per vjt's 2026-05-15 explicit "yes porco dio codify
that in claude.md" directive closes the loop — future-Claude six
months from now hits the closed-set fence before any code gets
written.

## Architecture (A1-A10 from the brainstorm)

- **A1 — Direct-to-litterbox.** Browser POSTs blob directly to
  `https://litterbox.catbox.moe/resources/internals/api.php`. The
  server never sees the bytes. Saves bandwidth + sidesteps any
  "image upload storage layer" obligation.
- **A2 — 📸-prefix wire shape.** PRIVMSG body is literally
  `📸 https://litter.catbox.moe/abc.png`. No IRC tags, no
  client-only namespace, no client-side detection magic.
- **A3 — Pluggable `ImageHost` interface.** The interface
  (`upload(blob, opts) → {url, expires_at}` + `name` +
  `default_ttl_seconds`) was validated against three hosters'
  docs: litterbox (TTL knob), 0x0.st (form-multipart), catbox-
  permanent (auth header). litterbox is the first concrete impl.
- **A4 — Per-host localStorage namespacing.** Privacy ack key is
  `image-upload-ack:<host-name>`. TTL preferences keyed identically.
- **A5 — CSP `connect-src` only, NOT `img-src`.** Cic doesn't
  render the image inline; the URL becomes a clickable link.
- **A6 — Four trigger surfaces.** 📸 compose button (desktop +
  mobile), mobile camera capture (`<input type=file
  capture=environment>` ≤768px), drag-drop onto compose textarea,
  clipboard paste. All four funnel through the same
  `imageUploadOrchestrator`.
- **A7 — Auto-send on resolve.** Orchestrator constructs the
  `📸 <url>` body and calls `compose.send` directly; operator's
  draft text in the textarea is preserved.
- **A8 — First-upload privacy modal.** Modal explains the host
  receives the image, lists the host's TTL, asks for ack. Per-host
  acks; subsequent uploads silent.
- **A9 — Clickable links via existing `linkify`.** No new render
  pathway; the URL is just a URL in a PRIVMSG body.
- **A10 — IRC stays text only.** Rule codified in CLAUDE.md.

## Lessons from the buckets

**1. CSP empirical pin: response host ≠ request host (I-CSP).**

The litterbox upload endpoint is `litterbox.catbox.moe`; the
successful response carries the URL on `litter.catbox.moe` —
note the dropped `box`. Both hosts must be in `connect-src`.
Captured empirically via curl during I-CSP verification — the
docs don't mention the host split.

**2. e2e selector strict-mode violation (I-2).**

`page.getByRole("dialog")` matched both the new `PrivacyModal`
AND the existing `SettingsDrawer` (which also carries
`role=dialog`). Playwright's strict mode rejected the ambiguous
locator. Fix: `page.getByRole("dialog", { name: /privacy/i })`.
Lesson: every future cic dialog needs an `aria-label` or visible
`<h2>` so e2e selectors have a name.

**3. Pre-session uncommitted state can block the deploy chain.**

A pre-cluster `cicchetto-build` oneshot had wiped
`runtime/cicchetto-dist/.gitkeep` while repopulating the dir with
bundle artifacts. The deletion sat as an unstaged `D` until I-CSP's
deploy chain hit `git pull --rebase` and refused to proceed.
Recovery: restore from HEAD. Mitigation TBD — either
`cicchetto-build` should preserve `.gitkeep`, or the dir should be
`.gitignored` entirely with the `.gitkeep` removed from tracking.
Captured but not actioned this cluster.
