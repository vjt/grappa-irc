The one real item: empty explicit-away reason. `set_explicit_away(_,_,"")`
slipped `safe_line_token?/1` (CR/LF/NUL-only) and emitted `AWAY :\r\n` —
the RFC 2812 §4.6 bare-AWAY **un-away** line — silently CLEARING away
instead of setting it. Reachable from a crafted WS push (`with_body_check`
only screens `body_too_large`).

Fixed at two layers: `Session.set_explicit_away/3,4` facade (primary,
covers both internal byte paths, rejects before `whereis`) **and**
`Client.send_away/2` byte boundary (defense-in-depth — completes the
`!= ""` symmetry `send_pong`/`send_raw`/`send_privmsg`/`send_part`/
`send_oper` already had; `send_away` was the lone exception the facade
docstring already *claimed* to mirror). Guard is `!= ""` NOT `String.trim`
(a spaces-only reason is a valid set, not the un-away line) — pinned by
test. cic `slashCommands.ts`: `/away :` now → `unset` (was `set`/`""`),
both empty cases collapsed. Bonus: fixed the pre-existing `compose.test.ts`
red above (api mock omitted `ChannelPushError` → `instanceof` threw on
every non-ApiError rejection).
