# CP15 — event-driven windows

Cluster opened 2026-05-07 immediately after CP14 closed at `620ef05`.
Implements the architectural shift pinned in
[`docs/plans/2026-05-07-event-driven-windows.md`](../plans/2026-05-07-event-driven-windows.md):
**one state model, on the server. cicchetto NEVER assumes state
transitions; it consumes typed JSON events.**

The premise: cic's optimistic mutation pattern (sidebar entries appear
on POST /channels success, members pane fetches once via REST and
caches) hides server reality. When IRC says invite-only, when upstream
goes parked, when the WS reconnect race drops the post-deploy
`members_seeded` broadcast, the UI silently lies — empty members,
ghost windows, scrollback rendered for channels we can't post in.

Window LIFECYCLE moves to event-driven (server emits `:joined`,
`:join_failed`, `:parted`, `:kicked`, `:window_state`). Window
CONTENT moves to event-driven (members + topic + modes already there;
cic's `loadedChannels` REST gate goes away). Window VISIBILITY stays
user-action-driven (PART/× moves to archive; KICK/T32 stays active in
greyed state).

Detailed bucket breakdown + TDD steps + decisions in
`docs/plans/2026-05-07-event-driven-windows-impl.md` (sibling, opens
with this checkpoint).

### B2 — server-side `:join_failed` event + in-flight JOIN map + failure numeric routing

**Goal**: on every outbound JOIN cic-initiated OR autojoin, record
`{channel, label?}` in `state.in_flight_joins`; on 471/473/474/475/
403/405 numerics, look up via labeled-response cap or channel-param
fallback, emit `:join_failed` carrying `{channel, reason, numeric}`,
broadcast on the per-channel topic.

**Status**: LANDED 2026-05-07.

`state.in_flight_joins :: %{lowercase_channel => {channel, at_ms,
label}}` populated by both the `Session.send_join/3` cast and the 001
RPL_WELCOME autojoin loop via the new `record_in_flight_join/2`
helper. Lazy O(1)-amortized 30s TTL sweep on insert keeps the map
bounded under upstream silence (no separate `Process.send_after`).
Lowercase key per RFC 2812 §2.2 so channel comparisons match
case-folded server echoes.

EventRouter clauses for the 6 failure numerics emit `{:join_failed,
channel, reason, code}` when `params[1]` matches an in-flight key,
and strip the entry from `next_state`. No-match falls through with no
effect — the existing `$server` route still surfaces the
server-emitted failure as a notice.

`apply_effects([{:join_failed, ...}])` arm in Session.Server writes
`window_states[ch] = :failed`, `window_failure_reasons[ch] = reason`,
persists a `:notice` scrollback row carrying `meta.numeric`, AND
broadcasts `kind: "join_failed"` on the per-channel topic.

NumericRouter `@delegated_numerics` extended to include the 6 codes so
the param-derived `$server` scan-route does NOT also persist the same
numeric — without delegation the same failure would surface twice.

Autojoin loop converted from `Enum.each` to a piped `Enum.reduce` so
state mutation threads through. The reduce → `maybe_fire_notify` →
`maybe_stage_pending_password` chain merges what was two `state =`
rebindings into one pipeline (Credo refactor flag for variable
shadowing).

### B3 — server-side `:parted`/`:kicked`/`:window_state` events + push_channel_snapshot extension

**Goal**: EventRouter emits `:parted` (own PART) + `:kicked`
(own-target KICK) effects in addition to the persist row; Session.Server
broadcasts both on the per-channel topic. `push_channel_snapshot/4` in
`lib/grappa_web/channels/grappa_channel.ex:637` extends to push the
cached `members_seeded` snapshot AND a fresh `window_state` event on
after_join (closes the deploy-reconnect race per intent decision #7).

**Status**: LANDED 2026-05-07.

EventRouter `:part` clause appends `{:parted, channel}` when sender ==
state.nick. `:kick` clause appends `{:kicked, channel, by, reason}`
when target == state.nick (`by` = sender nick; `reason` = trailing
param or nil). Other-actor PART/KICK keep the scrollback-only shape —
no window-state transition for events the operator isn't on the
receiving end of. Q1's uniform self-vs-other path preserved.

Session.Server apply_effects arms:
- `{:parted, ch}` drops `state.window_states[ch]` AND clears all four
  per-channel window-state mirror maps. Cic projects "no key +
  scrollback present" as `:archived`. Intentionally NO `kind: "parted"`
  broadcast — the `:persist :part` row in the same effects list is the
  UI feed-line; emitting a sibling event would force cic to dedupe.
- `{:kicked, ch, by, reason}` sets `state.window_states[ch] = :kicked`
  AND broadcasts `kind: "kicked"` carrying by + reason on the
  per-channel topic. The window stays in the active sidebar (greyed)
  so the operator can /join to retry; archiving on KICK would punish
  the victim.

Snapshot extension (the deploy-reconnect race fix) adds two private
helpers in `grappa_channel.ex` sibling to `push_topic_if_cached/4`:
- `push_members_if_seeded/4` re-emits the cached `members_seeded` list
  via `Session.list_members/3`. Empty list = skip (indistinguishable
  from "NAMES hasn't completed yet").
- `push_window_state_if_known/4` calls a new `Session.get_window_state/3`
  verb that returns the snapshot-ready event payload. Single source of
  truth: GenServer assembles the full payload (kind, network, channel,
  state, + optional reason / numeric / by) so the channel-side helper
  stays a one-liner. Payload is byte-identical to the corresponding
  event-time broadcast — cic's renderer does NOT branch on
  snapshot-vs-event origin.

To support i18n forward-compat (cic-side localized strings tomorrow,
server-provided strings today), state grows two new mirror maps:
- `window_failure_numerics: %{ch => 471|473|474|475|403|405}` — retains
  the failure numeric beyond event-time so the snapshot push carries
  it. Numerics are stable cross-language identifiers; reasons are
  upstream-language-locked. The future cic i18n layer maps numeric →
  localized template.
- `window_kicked_meta: %{ch => %{by, reason}}` — same motivation for
  KICK; cic shouldn't need a separate REST hop to render the kick
  banner on a deploy-reconnect.

apply_effects arms write these mirrors at event-time and clear them on
transitions out of `:failed` / `:kicked` (a successful re-JOIN wipes
stale by/reason/numeric so the next snapshot for that channel doesn't
lie).

T32 `:parked` window state lays the typespec slot but reads as
`:not_tracked` from the snapshot verb until disconnect verbs land —
keeps the surface forward-compatible without forcing cic to handle a
payload it doesn't yet render.

`:kick` clause cyclomatic complexity tipped from 9 to 10 with the
new effect-emission branch. Refactored in a separate commit
(`refactor(events): extract :kick clause helpers to drop CC below
gate`): `kick_state_update/3` + `evict_cache_if_no_overlap/3` private
helpers — pure refactor, no behavior change.
