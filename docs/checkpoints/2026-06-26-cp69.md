## Load-bearing gotchas (DO NOT relearn the hard way)

- **`wireTypes.ts` is GENERATED** (`mix grappa.gen_wire_types`, header says DO
  NOT EDIT). Any new `Session.Wire` payload ⇒ regenerate + commit it, or the
  `gen_wire_types --check` drift gate (in `ci.check`/`check.sh`) fails. **From a
  worktree the cic bind-mount is read-only — use `WRITABLE_CIC=1 scripts/mix.sh
  grappa.gen_wire_types`.** The codegen emits NESTED inline object fields at a
  fixed 2-space indent but biome wants 4-space ⇒ nested inline objects in a Wire
  typespec break BOTH gates at once. Fix = use a NAMED `@type` (done for
  `ChannelDirectory.Wire.entry`), never nested-inline.
- **Subject shapes:** `GrappaWeb.Subject.to_session(conn.assigns.current_subject)`
  → bare `{:user, uuid}` (feed BOTH `ChannelDirectory.list/3` and
  `Session.refresh_directory/2`). `Grappa.Subject` (`lib/grappa/subject.ex`) has
  `from_assigns/1`, `subject_where/2`, `put_subject_id/2` — there is NO
  `to_session` there (that's on `GrappaWeb.Subject`). The plan's
  `Grappa.Session.Subject` does not exist.
- **Gates are strict:** dialyzer enables `:underspecs` + `:unmatched_returns`
  (NO ignore file) — bind Repo bulk-op returns `{_, _} = Repo.x_all(...)`, spec
  constant-returning fns as `:: unquote(@const)` (Backoff/NetworkCircuit
  precedent). credo `--strict` wants BARE `_` for throwaways (NOT `_cred`/`_pid`
  — server_test.exs uses `_`). cic real type gate is `scripts/bun.sh run build`
  (`run check` = biome→tsc, but a biome failure SKIPS tsc). RUN credo+dialyzer
  before each Elixir commit; RUN `bun run build` after each cic commit.
- **Why C4 merged into C2:** `Session.Server` has no catch-all `handle_info`, so
  arming `:directory_refresh_timeout` without its handler crashes the session —
  timer + handler must ship together.
- **Boundary:** `ChannelDirectory` dirty_xrefs `Networks.Network` (NOT a full
  `Grappa.Networks` dep) to avoid a Networks→Session→ChannelDirectory cycle
  (mirrors Scrollback). `GrappaWeb` boundary gained `Grappa.ChannelDirectory`.
- Always `scripts/*.sh` — NEVER host `mix`/`bun`/`docker`. Subagent-driven:
  fresh implementer per task, then spec-review, then quality-review.
