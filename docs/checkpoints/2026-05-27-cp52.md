## S1 — 2026-05-27 — PWA manifest id + bastille cic-only deploy path

vjt reported an Android user seeing "developed for an earlier version
of Android and lacks the latest privacy protections" at PWA install
time, on a phone that had **never opened cic before**. Root cause:
Google's WebAPK Minting Server caches minted APKs keyed on the
manifest hash. An APK minted months ago with a now-stale
`targetSdkVersion` was being served to every new Android installer
of cic, triggering Play Protect's blanket block. The user had inherited
a stale APK someone else's install had originally caused to be minted.

Fix: add `id: "/cic"` to the W3C manifest (`cicchetto/vite.config.ts`).
The `id` field is a same-origin URL, never fetched — pure primary key
used by browsers + WebAPK minter to answer "is this the same app?".
Adding it shifts the manifest hash, forcing the minter to mint a fresh
APK with current targetSdkVersion. Existing installs are unaffected
because Chrome reconciles via `start_url` ↔ `id`. The comment in
vite.config.ts spells out the format + the never-change-this-after-
first-install rule so future edits don't orphan the install base.

The deploy revealed a substrate bug. Initial `jail_deploy_cic.sh`
(new script — first cic-only deploy path on the bastille jail,
parallel to `scripts/deploy-cic.sh` for Docker) ran clean but the
broadcast endpoint returned 204 "no bundle on disk." Investigation
showed `Grappa.Cic.Bundle.@bundle_path` hardcodes
`runtime/cicchetto-dist/index.html` (Docker compile-time anchor where
compose.yaml bind-mounts the container's `/app/dist`), but jail's
vite emitted to `cicchetto/dist/`. nginx served the new bundle fine
(it has its own symlink) but the refresh banner broadcast was silent.

Right fix: align the jail to the Docker substrate. `jail_cic_build.sh`
now passes `--outDir ../runtime/cicchetto-dist` to vite; nginx symlink
repoints at the shared path; `deploy.sh` cold path collapses to call
`jail_cic_build.sh` (DRY). One server-side anchor for both substrates,
zero per-substrate special-casing in `Grappa.Cic.Bundle`.

Verified live on `irc.sindro.me`: `id: "/cic"` present in manifest,
healthcheck green, refresh banner broadcast (hash `BydkWNJw`) reached
all connected user-topics. Two commits (`47f4bf0`, `bd1d85c`) pushed.
**BEAM never touched** — entire fix landed without restarting the
bouncer or disturbing any IRC session. First end-to-end exercise of
the cic-only deploy path on bastille.
