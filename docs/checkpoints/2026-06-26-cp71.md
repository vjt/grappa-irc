## Root cause (prod-confirmed)

`GET /networks/:net/channels` = `union(autojoin-source, live members)`. The
autojoin source diverges by subject:
- user → `Credential.autojoin_channels` (pruned by the DELETE controller),
- visitor → `Visitor.last_joined_channels`.

The cic × / `DELETE /channels` routes through
`Session.Server.handle_cast({:send_part, _})`, which dropped live members +
broadcast `channels_changed` **directly**, bypassing
`maybe_broadcast_channels_changed/2` — the only site that persisted the
`last_joined_channels` snapshot. So the snapshot stayed stale on every PART.
Masked for users (their source is `autojoin_channels`); fatal for visitors
(their source IS the stale snapshot) → parted channel kept rendering as
`source: autojoin, joined: false`, tab un-dismissable, re-× → 442. Both
subjects also rejoined the parted channel on reconnect (`state.autojoin` is
seeded from the snapshot via `merge_autojoin/2`).

Live prod query: visitor alk had `last_joined = ["#italia","#sniffo"]` while
live members were only `["#sniffo"]`.

## Fix (`d0868fd`) — one root, two doors, no second-class visitor

1. **`Session.Server`** — extracted `maybe_persist_last_joined/2`
   (+`channels_keyset/1`) as the single persister call site; routed BOTH the
   organic membership-change path and the `send_part` cast through it.
   Subject-agnostic. Closes case (a) — leaving a live-joined channel — for
   both subjects and kills the reconnect-rejoin. The cast keeps its
   unconditional `broadcast_channels_changed/1` (forces cic refetch even on
   a no-op eager wipe, per #38/UX-4-H).
2. **`Visitors.remove_autojoin_channel/2` + controller visitor branch**
   (was a no-op) — mirror of `Credentials.remove_autojoin_channel/3`. Closes
   case (b) — dismissing a stale autojoin entry the visitor is NOT
   live-joined to (e.g. all autojoin 475'd, no live membership to snapshot
   away).

Regression: 3 controller tests (user persist + visitor case a/b) + 4
`Visitors` context unit tests. DESIGN_NOTES entry (2026-06-26).
