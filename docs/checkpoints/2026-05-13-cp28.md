# CP28 — channel-state numerics (329/331/332/333/324) delegated, 329 wire event

Opened 2026-05-13. Brief originally at `/tmp/orchestrate-next.txt`
(channel-created-notice cluster). Discovered the plan's diagnosis was
wrong via live DB query — flagged back to vjt who reframed scope.

## Bug

`raccooncity.azzurra.chat` 333 RPL_TOPICWHOTIME landed as
`kind: notice, body: 1776720934` rows in scrollback (94 rows in the
dev DB). The unix timestamp leaked as user-visible noise. Same
disease for 332 RPL_TOPIC (94 rows duplicating the topic text already
conveyed by `topic_changed`).

## Root cause

`lib/grappa/session/numeric_router.ex @delegated_numerics` was
missing 329 / 331 / 332 / 333 / 324. Server's numeric handler
(`server.ex:1436`) persists every non-delegated numeric as a
`:notice` row BEFORE delegating to EventRouter. EventRouter's
dedicated handlers for 331/332/333/324 update `state.topics` /
`state.channel_modes` correctly — but the dual-path also wrote
duplicate `:notice` rows with body=trailing-param.

Plan claimed 329 was being silently dropped at the catch-all and
that Bahamut emitted bare-int NOTICE for channel-creation. Both
wrong:

- Live DB had ZERO 329 rows (Azzurra/Bahamut never emits 329).
- Live DB had ZERO bare-int NOTICE rows (no Bahamut bare-int NOTICE
  pattern exists).
- The "1776720934" noise was 333's trailing param, not a NOTICE.

## Fix shape

1. **Add `324, 329, 331, 332, 333` to `@delegated_numerics`** in
   `numeric_router.ex`. Stops Server from dual-persisting them as
   duplicate notice rows. EventRouter's existing handlers continue
   to populate state caches + broadcast typed wire events. Property
   test exclusion list updated.
2. **New 329 RPL_CREATIONTIME handler** in EventRouter. Caches
   `DateTime.t()` in new `state.channels_created` map, emits new
   `{:channel_created, channel, dt}` effect. Lifecycle mirrors
   `state.topics` (drop on self-PART, self-KICK).
3. **New `:channel_created` Server apply_effects clause** broadcasts
   on per-channel topic via `SessionWire.channel_created/3` (new
   wire helper). Wire shape carries ISO 8601 string.
4. **Cic-side**:
   - `narrowChannelEvent` adds `channel_created` arm.
   - `WireChannelEvent` union extended.
   - `channelTopic` store gains `createdByChannel` signal +
     `seedChannelCreated` setter.
   - `JoinBanner` renders 2 new lines:
     - "Channel was created on …" (from 329 cache)
     - "Topic set by … on …" (from existing 333-fed `set_by` /
       `set_at` — was already in store but never rendered)
