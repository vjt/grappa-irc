   release-helper pattern) + wired it into `scripts/deploy.sh`

**Severity counts:** 4 CRITICAL · 6 HIGH · 10 MEDIUM · 4 LOW.

**Three cross-cutting themes** surfaced from independent agents:

1. **Wire-format / topic-format helpers missing.** PubSub topic string is
   hand-interpolated in 2 broadcast sites + validated separately in channel
   guard + declared a 3rd time in router wildcards. Same shape for IRC command
   typing (closed set, but `String.t()`), event-payload helper (raw map
   construction at every call site), and Logger metadata (no canonical setter).
   Fix together as a "Phase 1.5 contract module" pass.

2. **Session.Server is over-broad.** Five findings on one module: mixes IRC
   framing with lifecycle, reaches into `to_wire/1` for broadcast,
   pattern-matches parser internal `{:nick, _, _, _}` tuple, accesses
   `Config.Network` struct fields directly, builds NICK/USER/PONG wire format
   inline. Phase 5 SASL/CAP will compound. Plan extraction of `Session.Protocol`
   submodule before Phase 2 SASL begins.

3. **Identifiers (`network_id`, `channel`, `sender`, `host`, `nick`) are
   unvalidated `String.t()`.** Two findings (Config + Scrollback) on the same
   structural pattern. Shared `Grappa.IRC.Identifier` module with regex
   constants + changeset validators.

**Verified strengths:** wire-shape unification (canonical case) works as
designed end-to-end; dependency direction is clean (web → contexts → schemas →
repo); `Application.fetch_env!` discipline upheld; no `try/rescue` in prod; test
discipline outcome-driven with real Repo + in-process IRC fake. Bootstrap
supervision strategy + migration future-proofing both vetted ✓.

**Recommended priority:** "Now" tier (~7h, two sessions) addresses themes #1
