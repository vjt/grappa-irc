## RC2 — m2 still RED (resume here tomorrow)

**Spec:** `m2-irssi-to-chan-defocused` — visit `#bofh` (200 seeded
backlog), defocus to $server, peer sends 1 PRIVMSG → expect badge "1",
**gets "201"** (200 backlog + 1).

**Root cause (confirmed via isolated m2 + grappa server logs):** the
read cursor is NEVER written for `#bofh` during m2 — zero
`read_cursors` INSERT/UPDATE, no `POST /read-cursor`. The leave-arm
(`ScrollbackPane.tsx` `on(key)` ~1190) computes
`id = visibleTailSnapshot.get(prevKey) ?? storeTail` and skips when both
are null. m2 switches away right after the own-JOIN line renders —
BEFORE the 200-msg REST backlog hydrates into `scrollbackByChannel` —
so `storeTail` is null and no snapshot exists → no cursor write → cursor
stays nil → server's nil-cursor `unread_count` = full backlog (200) +
the new msg = 201. `selectChannel`'s `awaitWsReady` only waits for the
JOIN line, not the backlog GET. (Contrast: `cursor-forward-only` waits
for rows, so its leave-arm has a tail → passes.)

**Planned fix (NOT yet implemented):** in
`cicchetto/src/lib/scrollback.ts` `loadInitialScrollback`, after merging
the loaded page, if `getReadCursor(slug, name) === null` advance the
cursor to the loaded tail (max id) via `setReadCursor` (forward-only
gate inline, as bucket D did). Rationale:
- "Opening a fresh channel marks its existing backlog as your read
  baseline" — a freshly-opened channel auto-scrolls to the bottom
  (newest), so cursor=tail is the honest "you've seen the newest."
- Gated on `cursor === null` → NEVER touches a channel that already has
  a read position, so the marker is preserved (consistent with vjt's
  "cursor-honest / leftover re-badges" answer; that scenario has a
  non-null cursor).
- Fires on load-COMPLETION → robust to the leave-race (the load was
  triggered by focus; completing after you leave still marks read).
- `loadInitialScrollback` only fires on focus, so unfocused new DMs
  (m4 "shows unread") are NOT auto-marked. Verify this holds.
- `scrollback.ts` already imports `get/setReadCursor` — no new cycle.
