## S3 ‚Äî 2026-06-21 ‚Äî PWA icon badge shipped + deployed to prod

The ONE task from the pending block below ‚Äî designed, implemented (9
phases, writing-plans ‚Üí TDD), shipped, and **deployed live to m42**.
Impl plan: `docs/plans/2026-06-21-pwa-icon-badge-impl.md`; full rationale
in DESIGN_NOTES 2026-06-21.

**Shape.** One predicate, three doors. `Grappa.Push.BadgeCount.count/1`
(0..99) reuses the REAL `Push.Triggers.should_notify?/4` over a bounded
unread tail (`Scrollback.unread_content_tail/6`, per-channel cap + 99
early-bail) ‚Äî no new persisted state, no second SQL copy of the notify
logic. Doors: push payload `badge`, `/me badge_count`, `read_cursor_set
badge_count`. cic `badge.ts` = one signal ‚Üí `setAppBadge` (feature-
detected) + `document.title` `(n)` mirror; SW stamps the icon on push
receipt even when the toast is suppressed.

**Two load-bearing structural calls** (both in DESIGN_NOTES):
- **Boundary inversion.** BadgeCount deps Networks/ReadCursor/Visitors,
  all of which transitively reach Session, and Session deps Push ‚Äî
  folding the counter into Push closes the cycle `Push ‚Üí Networks ‚Üí
  Session ‚Üí Push`. So BadgeCount is its OWN `top_level?: true` boundary
  ABOVE Push (same pattern as `Visitors.Reaper`). Door #1 (push hot path,
  below this layer) reaches it through a config-injected
  `Grappa.Push.BadgeSource` behaviour ‚Äî runtime resolution, no static
  `Push ‚Üí BadgeCount` edge.
- **own_nick off-Session.** Resolved from the configured credential nick
  (`Networks.configured_nick_index/1`), never `Session.current_nick` ‚Äî
  door #3 runs on the read-cursor settle hot path where a GenServer
  round-trip per network is unacceptable. Documented staleness after a
  `/nick` rename.

**Drift-proofing.** The cic `pushTriggers.ts shouldNotify` mirror and the
Elixir `should_notify?` both run a SHARED truth-table JSON fixture
(`cicchetto/src/lib/shouldNotifyTruthTable.json`) ‚Äî ExUnit + vitest
consume the identical cases (the test container bind-mounts cicchetto/src
so ExUnit reads the cic-side artifact). Add a branch ‚Üí add a row ‚Üí both
suites catch a divergence.

**Honest scope limit.** The foreground optimistic increment (so the
desktop title moves live on an unfocused mention) reuses the existing
mention-bump seam ‚Äî covers the channel-MENTION case (default-prefs notify
trigger). Non-mention triggers (channel-all / DM-all / whitelist) surface
on the next server sync, not optimistically, because cic has no global
notification-prefs signal. Count is server-authoritative throughout.

**Hot-deploy resilience.** A hot reload swaps code but does NOT re-run
`config.exs`, so `:badge_source` is briefly absent on a hot deploy.
`BadgeSource.count/1` returns `nil` in that window ‚Üí door #1 omits the
badge field (push still fires, icon untouched) rather than crashing the
fire-and-forget Task or stamping a wrong `0`. (Moot this deploy ‚Äî went
cold; see below.)
