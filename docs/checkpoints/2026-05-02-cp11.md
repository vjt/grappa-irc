### Recurring pattern: plan inherited a class of bugs

Both Task 1 and Task 2 surfaced the same bug class — plan-author
didn't grep the existing canonical validators (`Identifier.valid_*?/1`)
before writing Task 1 + Task 2 code blocks. Result: duplicated regex
with drift in Task 1 (caught by code-quality review C1); duplicated
length-only validation skipping syntactic shape in Task 2 (caught by
code-quality review's 3 Important). Both fixed by delegation to the
canonical source. Worth keeping watch for the same pattern across
Tasks 4–25.5 — anything that prescribes a fresh validator block
should be cross-checked against `Grappa.IRC.Identifier` + sibling
schemas before implementing.
