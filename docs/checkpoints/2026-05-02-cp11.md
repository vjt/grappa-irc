### Out-of-cluster baseline-hygiene commits on main (5)

User directive "fix tests flakiness porco dio" mid-Task-4 triggered a
pre-existing flake investigation. Pre-fix flake rate ~30% in
`scripts/test.sh`. Three distinct flake categories surfaced and were
addressed:

- `a4a56ae` + `e861f4c` — `Grappa.SessionSupervisor` `max_restarts`
  initial bump (3/5 → 100/60) + format follow-up. Closed BootstrapTest
  on_exit cascade where the supervisor's GLOBAL `max_restarts` budget
  was being exhausted by routine teardown crashes (linked IRCServer
  fakes dying with the test PID → IRC.Client `:tcp_closed` → Session
  `:transient` restart → connect-refused → loop).
- `6a8888d` — `IRCServer.accept` timeout 5s → 30s. Pre-fix the test
  helper's acceptor process died silently after 5s, causing subsequent
  Session connect attempts to hit `:econnrefused` even when the test
  was still actively setting up. Test-support only; production
  unaffected. Documented inline rationale.
- `1019fbd` — `Grappa.SessionSupervisor` second bump (100/60 → 10_000/60).
  Detour through `ef4bf62` (1s `Process.sleep` in `IRC.Client.handle_continue`
  before `:connect_failed` exit, attempting to rate-limit the restart
  loop at the source) BROKE the C2 contract test
  (`assert_receive {:EXIT, ^client, {:connect_failed, :econnrefused}},
  1_000` raced against the new sleep). Reverted in `1019fbd` and
  bumped the supervisor budget to 10_000/60 instead, which absorbs
  ~5 seconds of full-rate (~2000 crashes/sec) restart loop before
  tripping. DESIGN_NOTES 2026-05-02 entry updated to record the
  detour so future Claude doesn't re-walk it.

Post-fix flake rate ~20% — improvement but not zero. The remaining
20% is sqlite `Database busy` on Users INSERT in test setup callbacks
(Argon2 hashing + parallel `max_cases: 2` writers + 30s `busy_timeout`
already at the high end). Deeper fix would require either
`async: false` on every test that hashes a password OR migrating off
sqlite. NOT addressed in this session — flagged as future work, may
not resolve before visitor-auth ships.
