## What landed this session

### Log routing under runtime/

After two passes, the final on-disk layout in prod:

```
runtime/
├── log/erlang.log.*  ← run_erl tee of BEAM stdout (RELEASE_TMP=runtime)
├── pipe/erlang.pipe.1.{r,w}  ← bin/grappa remote attach pipe
├── grappa_prod.db (+ -shm + -wal)
├── uploads/
├── bun-cache/
└── cicchetto-dist/
```

Single source of on-disk log: `run_erl`'s tee in prod, Docker
json-file driver in dev. No parallel Elixir-side file sink (first
pass tried that; reverted — produced same lines in two paths).

Driven by `RELEASE_TMP=/home/grappa/grappa/runtime` exported by
`infra/freebsd/rc.d/grappa`'s `grappa_runas/1`. RELEASE_TMP one
level UP from `runtime/log` because run_erl always creates its
own `log/` and `pipe/` subdirs.

### Bugs caught + landed even after the file-sink revert

- `mix release` CWD ≠ repo root → relative paths in `runtime.exs`
  `:eacces` under grappa user. `config/runtime.exs` defaults now
  derive from `Path.dirname(database_path)` so anything keyed on
  on-disk state inherits the already-absolute env-driven root.
- `RELEASE_TMP='...' . envfile && cmd` POSIX gotcha: the prefix
  binding dies with the `.` source builtin. Fixed by `export
  RELEASE_TMP;` as a separate statement.

### CI side-fix

`admin_events_test.exs` setup flunked 10 consecutive times on the
GHA runner (`SessionRegistry never drained — stale entries:
[...]`). Same rotating-cascade pattern as
`feedback_ci_cascade_rotating_set`: green locally, fails under
coveralls load. `Session.stop_session/2` returns once the worker
pid is dead but Registry's monitor-DOWN cleanup runs async. 50ms
single-sleep was too tight; replaced with 200×10ms poll matching
the upstream passive-wait shape (commit 1108808).
