## M-Z deliverable: cross-cluster e2e spec

Per orchestrator vjt 2026-05-16: ad-hoc browser walk-throughs at
cluster close aren't reproducible. M-Z ships a proper Playwright
spec under `cicchetto/e2e/tests/m-z-admin-cluster-journey.spec.ts`
that runs in CI on every integration run, forever.

Spec exercises the full plan §M-Z 7-step operator journey end-to-end:

1. Login as admin → drawer entry visible (M-7 gate).
2. Open Visitors tab → asserts table OR explicit empty marker visible.
3. Open Sessions tab → asserts ≥ 1 seeded row.
4. Open Networks tab → asserts bahamut-test + azzurra rows visible.
5. Open Events tab → mounts cleanly.
6. Cap-saturation arm: PATCH azzurra cap to 0, attempt visitor mint
   (expected 503 `network_busy`), assert typed `capacity_reject`
   admin event row lands in Events tab via the
   `grappa:admin:events` channel in real time. Revert in `finally`
   so the seeder's baseline cap (100) is restored.
7. Non-admin gate parity — covered by existing
   `m7-admin-gate.spec.ts`; M-Z doesn't duplicate.

The cap-saturation arm is the critical assertion — it validates
the END-TO-END M-11 wire from controller → admission telemetry →
AdminEvents.record → broadcast → cic channel ingest → list render
under live WS conditions, in 8 seconds.

## Architectural calls collected (cluster summary)

Lifted into `docs/DESIGN_NOTES.md` "M cluster CLOSED" entry + 2
new CLAUDE.md rules at M-12. Highlights:

- **User-rooted topics + dedicated admin topic.** `grappa:user:{name}`
  for per-user; `grappa:admin:events` sibling top-level for admin
  fan-out. Single source of truth `Grappa.PubSub.Topic`.
- **WS-boundary authz at `join/3`, never per-message.** M-11
  reviewer-caught CRIT-1: pre-fix gated in `handle_in/3`, would
  have allowed non-admin sockets to subscribe to admin events.
- **`Grappa.AdminEvents` singleton ring buffer cap=200.**
  Diagnostic tail, not audit trail. Persistent log evaluated +
  rejected (state-changing endpoints already DB-source-of-truth).
- **InlineConfirmButton lifted at 2nd use case** (M-9b session
  disconnect after M-8 visitor delete); 3rd + 4th use cases (M-10
  Reset Circuit + Force Reap) validated the boundary held.
- **Composite-id URLs for `/admin/sessions/:id`** —
  `"kind:uuid:network_slug"`. Sessions are per-`(subject,network)`;
  composite avoids a parallel routing-table mirror.
- **Two-tier identity flows through admin endpoints uniformly** —
  visitor-disconnect collapses to terminate (visitors have no
  parked state); branching is controller-side, URL-shape-agnostic.
- **DB state + live state are separate sources of truth.**
  `AdminSessionsTab` surfaces BOTH `connection_state` (DB) and
  `live_pid` (registry lookup); `null` is the U-0 honesty signal.
  Now a CLAUDE.md rule (M-12).
- **`/admin/<resource>` requires nginx allowlist** in both
  `infra/nginx.conf` + `cicchetto/e2e/nginx-test.conf`, both
  `:80` + `:443` blocks. Now a CLAUDE.md rule (M-12). Origin:
  latent bug surfaced at M-9b.
